# Dotconfig
This repo stores all my config for different clis

## Installation
- Clone the repo to the ~/.config directory
- `git submodule update --init --recursive`

## NVIM
My heart, my soul and my passion
- Native Neovim v0.11 lsp support.
- Still rocking the treesitter, dun have time to upgrade to Neovim v0.14.
- Favorite plugins:
  - `mini.surround` -> simple concept, supreme execution. Saved me thousands of seconds at the very least
  - `inc-rename.nvim` -> again very simple concept, but makes my life tiny bit better a thousand times.
  - `snacks.nvim` -> my "oh my god i can configure this" moment, i know fff.nvim probably has better performance, but who cares, i am enjoying my snacks!
  - `oil.nvim` -> i wish finder has nvim binding, oil.nvim: hold my oil.
  - `grug-far` -> surprisingly useful, cross file search, replace some / all matches. 
  - `day-record-plugin` -> my first nvim plugin, completely vibe coded it, and i love it to depth. Completely customized to my likings and workflows.

## Opencode
### Plugins
- env-protection.js -> make sure the agent are not reading the env var 99% of the time.
- usage-tracker.js -> homemade usage tracker, backfill, event replay, rollups. Data in turso served with [opencode-stats](https://github.com/wcleeah/opencode-stats)
### Skills
- `deep-dive-research` -> skill for doing a deep dive research on a certain topic.
- `implementation-plan` -> create a implementation plan with a summarised context.
- `phase-execution-briefs` -> create agent native phased execution md based on the implementation plan, with concrete entry and exit gates.
